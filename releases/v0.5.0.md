# ALP v0.5.0 — Proxy Tool Execution + MCP SSE Transport

**Released:** 2026 · Backward-compatible with v0.4.0. No card schema changes.

## What's new

### Proxy Tool Execution

When a tool's `endpoint` in `agent.alp.json` is a full URL, the ALP Server
forwards the call to that URL via HTTP — automatically, with no code changes
needed in the remote agent.

```json
{
  "name": "search",
  "description": "Search the knowledge base.",
  "endpoint": "https://their-server.com/api/search"
}
```

The ALP Server POSTs `{"input": {...}}` to the endpoint and returns the result
to the caller. Errors from the remote endpoint are surfaced cleanly.

- Relative endpoint (`/tools/search`) → executes locally (existing behaviour)
- Full URL endpoint (`https://...`) → proxied to the remote agent (new)

This means any existing agent with HTTP tool endpoints can be loaded into
Kiro or any MCP runtime by adding one `agent.alp.json` file — zero code
changes to their agent.

### MCP SSE Transport

The reference server now exposes a full MCP SSE transport layer:

| Endpoint | Method | Description |
|---|---|---|
| `/mcp` | GET | SSE stream — Kiro connects here (`text/event-stream`) |
| `/mcp` | POST | MCP JSON-RPC message receiver |

Kiro MCP config (`.kiro/settings/mcp.json`):

```json
{
  "mcpServers": {
    "my-agent": {
      "url": "http://localhost:8000/mcp"
    }
  }
}
```

Supports: `initialize`, `tools/list`, `tools/call`, `notifications/*`.

### New dependency

```
httpx>=0.27.0
```

Added to `reference/server/python/requirements.txt` for proxy HTTP calls.

## Migration from v0.4.0

No card schema changes. All v0.4.0 cards work without modification.

To upgrade:
- Replace `reference/server/python/alp_server.py` with the v0.5.0 version
- Run `pip install -r requirements.txt` to install `httpx`
- Change `alp_version` in your card to `"0.5.0"` (optional)
- For Kiro: point `.kiro/settings/mcp.json` at `/mcp` instead of `/tools`
