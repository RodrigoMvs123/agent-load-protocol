# ALP v0.2.2 — Workforce, Tool Steps, Alerts, Bulk Schedule

**Released:** 2024 · Backward-compatible with v0.2.1.

## What's new

### `workforce` block
Models multi-agent teams. Each card describes its role and outbound connections.

```json
"workforce": {
  "role": "manager",
  "connections": [
    { "target_agent_id": "writer-agent", "connection_type": "next-step", "label": "Send draft to writer" },
    { "target_agent_id": "reviewer-agent", "connection_type": "ai", "condition": "draft_ready == true" }
  ]
}
```

Connection types: `ai` (LLM decides when to hand off), `next-step` (fires automatically on completion).
Roles: `standalone` (default), `manager`, `worker`, `reviewer`.

### `tools[].steps` + `tools[].output_schema`
Tools can now declare an internal pipeline of steps. Each step can use the output of previous steps.

Step types: `llm_prompt`, `api_call`, `code`, `integration`, `knowledge_search`.

### `knowledge[].retrieval_mode`
Controls how knowledge is surfaced to the agent:
- `add_all_to_prompt` — entire set included every run (best for small datasets)
- `allow_agent_to_search` — RAG-based retrieval (best for large datasets, default)

### `alerts` array
Agents can notify humans or escalate on events.

```json
"alerts": [
  { "trigger": "human_approval_required", "action": "pause", "channel": "slack", "auth_ref": "slack_webhook_url" },
  { "trigger": "tool_failure", "action": "notify", "channel": "email", "auth_ref": "alert_email_address" }
]
```

Actions: `notify`, `escalate`, `pause`, `stop`.

### `bulk_schedule` block
Run an agent across a list of inputs in bulk.

```json
"bulk_schedule": {
  "enabled": true,
  "input_source": "knowledge_table",
  "input_source_ref": "leads_table_id",
  "concurrency": 5
}
```

### `server.region`
Region-scoped API endpoints (e.g. Relevance AI).

```json
"server": { "url": "https://api-f1db6c.stack.tryrelevance.com/latest/...", "region": "f1db6c" }
```

### Secrets template syntax
Variables block now supports `{{secrets.key_name}}` pattern for tool step injection.

## New capabilities

`workforce-member`, `bulk-execution`, `alerting`, `rag`

## Migration from v0.2.1

All v0.2.1 cards are valid v0.2.2 cards. All new fields are optional. No breaking changes.

To upgrade: change `alp_version` to `"0.2.2"`.
