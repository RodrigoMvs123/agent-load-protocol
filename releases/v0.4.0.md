# ALP v0.4.0 — Server Contract, Portability Layer, Node.js Reference

**Released:** 2024 · Backward-compatible with v0.3.0. No card schema changes.

## What's new

### `GET /persona` — new required server endpoint
Returns the agent's persona text so any runtime can inject it as a system prompt before registering tools.

```
GET /persona
→ { "persona": "You are a helpful assistant...", "id": "my-agent", "name": "My Agent" }
```

### `GET /agents` — new optional server endpoint
Returns all Agent Cards hosted by a server. When `AGENTS_DIR` is set, the server scans that directory for `agent.alp.json` files. Falls back to the single card at `AGENT_CARD_PATH`.

```
GET /agents
→ { "agents": [ { "id": "code-reviewer", ... }, { "id": "sales-researcher", ... } ] }
```

Marketplaces (e.g. HoneyComb) use this to display a full catalog without running any agent.

### Node.js / TypeScript reference server
`reference/server/node/alp_server.ts` — a TypeScript port of the Python server using Fastify. Feature-identical: all endpoints, same env vars, same behaviour.

```bash
cd reference/server/node
npm install
AGENT_CARD_PATH=../../../examples/hello-agent/agent.alp.json npm start
```

### `SECRETS.md` — definitive secrets guide

## Migration from v0.3.0

No card schema changes. All v0.3.0 cards work without modification.

Server implementations should add:
- `GET /persona` — required in v0.4.0
- `GET /agents` — optional, recommended for multi-agent deployments

To upgrade your card: change `alp_version` to `"0.4.0"` (optional).
