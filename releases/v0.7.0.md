# ALP v0.7.0 — Middleware Library (`pip install alp-server`)

**Released:** 2026 · Backward-compatible with v0.6.0. No card schema changes.

## What's new

### `pip install alp-server`

The ALP SSE + MCP transport layer is now an installable Python library.
Any existing FastAPI or Flask server adds full MCP/Kiro connectivity in 3 lines —
no separate server process, no cloning the ALP repo.

```bash
pip install alp-server
```

### FastAPI — 3 lines

```python
from fastapi import FastAPI
from alp import ALPRouter

app = FastAPI()
alp = ALPRouter(card_path="agent.alp.json")
app.include_router(alp.router)
```

Adds: `/mcp`, `/agent`, `/agent/refresh`, `/tools`, `/persona`, `/agents`, `/health`

### Flask — 3 lines

```python
from flask import Flask
from alp.flask import ALPBlueprint

app = Flask(__name__)
alp = ALPBlueprint(card_path="agent.alp.json")
app.register_blueprint(alp.blueprint)
```

### `@alp.tool()` decorator

Register local Python functions as tool handlers:

```python
@alp.tool("search")
async def search(input_data: dict) -> dict:
    return {"results": my_search(input_data["query"])}
```

### Package structure

```
alp-server/
├── alp/
│   ├── __init__.py     ← ALPRouter export
│   ├── card.py         ← Card loader (local + remote, v0.6.0)
│   ├── tools.py        ← Tool registry + proxy executor (v0.5.0)
│   ├── mcp.py          ← MCP JSON-RPC handler
│   ├── sse.py          ← SSE transport layer
│   ├── fastapi.py      ← FastAPI router
│   └── flask.py        ← Flask blueprint
├── pyproject.toml
└── README.md
```

### Proxy and remote card support included

- Proxy tool execution from v0.5.0 works via the library
- Remote card loading from v0.6.0 works via `card_url` parameter or `AGENT_CARD_URL` env var

## Migration from v0.6.0

No breaking changes. The reference server (`reference/server/python/alp_server.py`) is unchanged.

The library is a new addition — existing setups continue to work exactly as before.

To use the library in your own project:

```bash
pip install alp-server
```
