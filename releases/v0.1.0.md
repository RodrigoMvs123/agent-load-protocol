# ALP v0.1.0 — Initial Release

**Released:** 2024 · First published version of the Agent Load Protocol.

## What is ALP

ALP defines a single portable artifact — the **Agent Card** (`agent.alp.json`) — that fully describes any AI agent: its identity, persona, tools, memory configuration, and LLM preference. Any runtime that speaks ALP can load any agent that ships an Agent Card.

## What's in this release

### Agent Card fields

| Field | Required | Description |
|---|---|---|
| `alp_version` | ✅ | Protocol version — must be `"0.1.0"` |
| `id` | ✅ | Unique agent identifier |
| `name` | ✅ | Human-readable name |
| `description` | — | Short description |
| `persona` | ✅ | System prompt / agent identity |
| `llm` | ✅ | LLM preference (`provider`, `model`) |
| `tools` | — | Array of tool definitions |
| `memory` | — | Memory config (`enabled`) |
| `server` | ✅ | Server URL and transport |
| `metadata` | — | Author, version, tags |

### ALP Server endpoints

| Endpoint | Method | Description |
|---|---|---|
| `/agent` | GET | Returns the Agent Card JSON |
| `/tools` | GET | Returns list of tools |
| `/tools/{name}` | POST | Executes a named tool |
| `/health` | GET | Returns `{"status": "ok"}` |

### Reference implementation

- Python server (`reference/server/python/alp_server.py`) using FastAPI
