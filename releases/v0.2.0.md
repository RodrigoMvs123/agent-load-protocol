# ALP v0.2.0 — Agent Types, Capabilities, Observability, Marketplace

**Released:** 2024 · Backward-compatible with v0.1.0.

## What's new

### `agent_type` field
Describes the topology of the agent. Optional, defaults to `single`.

| Value | Description |
|---|---|
| `single` | Standalone agent with one identity and one tool set |
| `multi-agent` | Coordinated graph of agents with defined roles |
| `swarm` | Dynamically spawned worker pool (e.g. Hive queen + workers) |

### `capabilities` array
Declares what the agent can do at runtime. Used by clients and marketplaces to filter agents without running them.

```json
"capabilities": ["tool-use", "web-search", "streaming"]
```

Supported values: `human-in-the-loop`, `self-healing`, `parallel-execution`, `streaming`, `memory`, `tool-use`, `code-execution`, `web-search`.

### `observability` block
Describes how to stream or monitor agent execution.

```json
"observability": {
  "websocket": true,
  "endpoint": "/stream",
  "logs_endpoint": "/logs"
}
```

### `marketplace` block
Metadata for agent discovery platforms (e.g. HoneyComb). Lets a marketplace render a full listing from the card alone — without running the agent.

```json
"marketplace": {
  "category": "automation",
  "tags": ["business-process"],
  "pricing_model": "per-run",
  "icon_url": "https://example.com/icon.png"
}
```

### `auth_ref` on tools
Tools that require credentials now use `auth_ref` — a string name the runtime resolves locally. The card never contains actual secrets.

```json
{
  "name": "search",
  "endpoint": "/tools/search",
  "auth_ref": "serper_api_key"
}
```

## Migration from v0.1.0

All v0.1.0 cards are valid v0.2.0 cards. All new fields are optional. No breaking changes.

To upgrade: change `alp_version` to `"0.2.0"`.
