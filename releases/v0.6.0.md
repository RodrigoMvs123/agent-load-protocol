# ALP v0.6.0 — Remote Card Loading + Multi-Agent Manifest

**Released:** 2026 · Backward-compatible with v0.5.0. No card schema changes.

## What's new

### Remote Card Loading (`AGENT_CARD_URL`)

The ALP Server can now load an `agent.alp.json` directly from any public URL
at startup — no local file needed. Point it at a GitHub raw URL and the server
serves `/mcp` on behalf of that agent.

**macOS / Linux:**
```bash
AGENT_CARD_URL=https://raw.githubusercontent.com/their-org/their-repo/main/agent.alp.json \
  python alp_server.py
```

**Windows PowerShell:**
```powershell
$env:AGENT_CARD_URL = "https://raw.githubusercontent.com/their-org/their-repo/main/agent.alp.json"
python alp_server.py
```

Falls back to `AGENT_CARD_PATH` (local file) if `AGENT_CARD_URL` is not set.
No breaking change.

### `GET /agent/refresh` — live card reload

Re-fetches the card from `AGENT_CARD_URL` without restarting the server.
Useful for deployments where the card is updated frequently.

```
GET /agent/refresh
→ { "refreshed": true, "id": "...", "name": "...", "alp_version": "...", "source": "..." }
```

### Multi-Agent Manifest (`AGENTS_MANIFEST`)

A single ALP Server instance can now host a catalog of remote agents.
Point `AGENTS_MANIFEST` at a JSON file listing public card URLs:

```json
{
  "agents": [
    "https://raw.githubusercontent.com/org-a/repo/main/agent.alp.json",
    "https://raw.githubusercontent.com/org-b/repo/main/agent.alp.json"
  ]
}
```

**macOS / Linux:**
```bash
AGENTS_MANIFEST=examples/remote-card/agents.json python alp_server.py
```

**Windows PowerShell:**
```powershell
$env:AGENTS_MANIFEST = "examples\remote-card\agents.json"
python alp_server.py
```

Each agent is loaded at startup and cached in memory by its `id` field.

### Per-agent MCP endpoints

Each agent loaded from the manifest gets its own MCP SSE endpoint:

```
GET  /mcp/{agent_id}   → SSE stream for that agent
POST /mcp/{agent_id}   → JSON-RPC receiver for that agent
```

Kiro multi-agent config (`.kiro/settings/mcp.json`):

```json
{
  "mcpServers": {
    "agent-a": { "url": "http://localhost:8000/mcp/agent-a" },
    "agent-b": { "url": "http://localhost:8000/mcp/agent-b" }
  }
}
```

### New example: `examples/remote-card/`

`examples/remote-card/agents.json` — a ready-to-use manifest file pointing
at the `alp-agent-starter` card on GitHub as a reference.

## Migration from v0.5.0

No card schema changes. All v0.5.0 cards and server configs work without modification.

To upgrade:
- Replace `reference/server/python/alp_server.py` with the v0.6.0 version
- No new dependencies — `httpx` was already added in v0.5.0
- `AGENT_CARD_URL` and `AGENTS_MANIFEST` are optional — existing local setups unchanged
- Change `alp_version` in your card to `"0.6.0"` (optional)
