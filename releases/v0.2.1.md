# ALP v0.2.1 — Variables, Triggers, Knowledge, Platform Origin

**Released:** 2024 · Backward-compatible with v0.2.0.

## What's new

### `variables` block
Template variables injected into the persona and tool prompts at runtime (e.g. `{{repo_path}}`). The card declares what variables exist; the runtime resolves their values.

```json
"variables": {
  "repo_path": { "description": "The GitHub repo path.", "required": true },
  "main_branch_name": { "description": "Main branch name.", "required": true, "default": "main" }
}
```

### `triggers` array
Agents can be started automatically by events — webhooks, schedules, emails.

```json
"triggers": [
  { "type": "webhook", "config": { "path": "/trigger/inbound" } },
  { "type": "schedule", "config": { "cron": "0 9 * * 1-5" } },
  { "type": "email", "config": {} }
]
```

Supported types: `webhook`, `schedule`, `email`, `manual`, `event`.

### `knowledge` array
Documents and data sources attached to the agent. Credentials use `auth_ref`.

```json
"knowledge": [
  { "id": "api-docs", "type": "document", "description": "API reference." },
  { "id": "db", "type": "database", "description": "Internal DB.", "auth_ref": "db_connection_string" }
]
```

### `platform` block
Records where the agent was built. Informational — does not affect loading.

```json
"platform": {
  "name": "relevance-ai",
  "agent_id": "bcbe5a-...",
  "project_id": "my-project-id",
  "export_url": "https://app.relevanceai.com/agents/..."
}
```

## New capabilities

`github-integration`, `knowledge-retrieval`, `triggered-execution`

## Migration from v0.2.0

All v0.2.0 cards are valid v0.2.1 cards. All new fields are optional. No breaking changes.

To upgrade: change `alp_version` to `"0.2.1"`.
