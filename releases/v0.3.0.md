# ALP v0.3.0 — Toolsets, Security, Dynamic Discovery, Server Manifest

**Released:** 2024 · Backward-compatible with v0.2.2.

## What's new

### `toolsets` block
Group tools into named sets. Clients enable or disable a whole set at once, reducing LLM context size.

```json
"toolsets": {
  "groups": {
    "default": ["github_api_call", "create_branch", "create_pull_request"],
    "readonly": ["github_api_call"]
  },
  "active": "default"
}
```

### `security` block — read-only mode + lockdown
`read_only: true` strips all mutating tools automatically. `lockdown_mode: true` filters tool outputs from untrusted sources to defend against prompt injection.

```json
"security": { "read_only": false, "lockdown_mode": false }
```

Tools declare their mutability with `tools[].readonly`.

### Per-tool OAuth scopes
Each tool can declare exactly which OAuth scopes it requires — required for marketplace trust.

```json
{
  "name": "create_pull_request",
  "auth": { "type": "oauth", "required_scopes": ["repo"], "accepted_scopes": ["repo", "public_repo"] }
}
```

Auth types: `bearer_token`, `oauth`, `api_key`, `none`.

### `tools_discovery` — dynamic tool discovery
When enabled, the LLM can discover and enable tools at runtime based on what the task needs.

```json
"tools_discovery": { "enabled": true, "mode": "dynamic" }
```

New endpoints when dynamic: `GET /tools/discover`, `POST /tools/enable`, `GET /tools/list`.

### `pagination` block
Standard pagination settings for tools that return lists.

```json
"pagination": { "style": "offset", "default_page_size": 30, "max_page_size": 100 }
```

### Tool aliases + deprecation
Renamed tools preserve old names as aliases so existing integrations don't break.

```json
{ "name": "github_api_call", "aliases": ["github_call", "gh_api"], "deprecated": false }
```

### Tool description overrides
Override tool descriptions via env var without changing the card — useful for localization.

```json
{ "name": "github_api_call", "description_override_key": "TOOL_GITHUB_API_CALL_DESCRIPTION" }
```

### `server.channel` — insiders / preview
```json
"server": { "channel": "stable", "insiders_url": "https://your-server.com/insiders" }
```

### `server.alp.json` — server manifest
A machine-readable manifest at the server root. Clients read it before connecting to negotiate capabilities.

## Migration from v0.2.2

All v0.2.2 cards are valid v0.3.0 cards. All new fields are optional. No breaking changes.

New top-level fields: `toolsets`, `tools_discovery`, `pagination`, `security`.
New tool-level fields: `readonly`, `deprecated`, `aliases`, `replaces`, `auth`, `description_override_key`.
New server fields: `channel`, `insiders_url`.
New artifact: `server.alp.json`.

To upgrade: change `alp_version` to `"0.3.0"`.
